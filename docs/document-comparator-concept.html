Document Version Comparator

Concept Architecture — compare against implementation

Layer 1 — Input
Version A (Old)
Drag & drop or click to upload
File name + size displayed on select
PDF DOCX XLSX / XLS / CSV TXT
Version B (New)
Same format as A recommended
Cross-format comparison supported
PDF DOCX XLSX / XLS / CSV TXT

Both files processed entirely in-browser. No data transmitted or stored.

Layer 2 — Extraction
PDF pdfjs-dist
1. getTextContent() per page
2. Word count check
3. < 20 words → OCR fallback
Text output
Scanned PDF Tesseract.js OCR

OCR per page with progress %
Language: English (eng)
Text output
DOCX mammoth
mammoth.extractRawText()
Accepted text only
Tracked changes excluded
Text output
Excel SheetJS
XLSX.read() → per-sheet cell map
Prefers formula (=) over value
Address: A1, B2 notation
Structured output

Extraction path detected from file extension. PDF auto-switches to OCR if native text is insufficient.

↓
Layer 3 — Diff Engine
Text Path (PDF / DOCX / TXT)
lineDiff(oldText, newText)
Split into non-empty lines/paragraphs
LCS algorithm (Myers-style DP table)
Adjacent removed+added → paired as modified

wordDiff(oldStr, newStr)
Run on modified pairs only
Produces: equal / added / removed word tokens
Rendered as inline highlighted spans
Excel Path
excelDiff(oldSheets, newSheets)
Compare cell-by-cell per sheet
Detect: added cell / removed cell / changed cell
Sort by row then column (A1 → Z999)
Sheet-level: added sheet / removed sheet / changed

Limitation: Row insertion shifts addresses.
No automatic row-key matching in v1.

Output: array of change objects [{type, old, new, tags, wordDiff?}] for text; {sheetName: [{cell, type, old, new, tags}]} for Excel.

↓
Layer 4 — Audit Classification (runs on every changed item)
Control Weakened
Old: shall / must / is required to / will be
New: should / may / can / is recommended
Binding → permissive shift
Control Strengthened
Old: should / may / can
New: shall / must / is required
Permissive → binding shift
Threshold Change
Any numeric value differs between old and new
Captures: amounts, %, headcount, days, limits
Responsibility Scope
Resp: director / manager / board / head / officer keyword
Scope: includes / excludes / applies to / exempt

Multiple tags can apply to a single change. Tags drive the Critical filter in the display layer.

Limitations — displayed at top of every result
  • OCR (Tesseract.js): unreliable on two-column layouts, tables, headers/footers, low-DPI scans, handwriting
  • Excel: row/column insertion shifts cell addresses — results may show false positives in dense tables
  • DOCX: tracked changes (Track Changes) are excluded — only the accepted final text is compared
  • Large files (>10 MB PDFs, scans with many pages): OCR is slow on user device — progress shown
  • No export: use browser copy or screenshot for working papers
